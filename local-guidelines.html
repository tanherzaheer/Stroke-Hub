Local Guidelines

This page is a placeholder for department- or trust-specific stroke protocols. It is intended to be filled in by your local stroke team.

About this page

Every hospital, trust, or region has its own stroke pathway — thrombolysis inclusion/exclusion criteria, thrombectomy referral routes, on-call contact numbers, and local prescribing protocols will all differ. Rather than guess at content that could be wrong for your setting, this page is left as a clearly-marked placeholder for your local team to complete.

Suggested structure for local content

When populating this page, consider including:

  • Hyperacute pathway: local thrombolysis and thrombectomy criteria, referral pathway, and target times (e.g. door-to-needle).
  • Key contacts: stroke on-call number, thrombectomy centre transfer arrangements, neurosurgical on-call contact.
  • Thrombolysis checklist: local inclusion/exclusion criteria and consent process.
  • Imaging pathway: how to request and prioritise urgent CT/CTA/CTP or MRI out of hours.
  • TIA clinic details: referral route, opening hours, and expected turnaround time.
  • Anticoagulation reversal protocol: local availability and dosing of reversal agents.
  • Rehabilitation and discharge pathway: local stroke unit, early supported discharge team, and community follow-up.
  • Links to trust intranet documents or externally hosted local protocols, where these cannot be reproduced here.

How to add a new page like this one

Stroke Hub is a static site built with plain HTML, CSS, and a small amount of JavaScript, designed so new topic or local pages are easy to add:

  1. Copy an existing page (e.g. this one) as a starting template — it already includes the shared header, navigation, footer, and styling.
  2. Update the <title>, meta description, page heading, and content sections.
  3. Add a link to the new page in the <nav> menu of every page (or, for local sub-pages, link from this page instead of the main navigation).
  4. Keep section headings wrapped in <section id="..."> elements so a table of contents can be generated if needed, and so the content stays easy for search engines and readers to scan.
  5. Commit and push — GitHub Pages will redeploy automatically.
